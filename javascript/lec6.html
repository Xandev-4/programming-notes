Mastering Functions in JavaScript

Declarations, expressions, parameters, return, defaults, rest, nested, callbacks, pure, HOF, arrow functions, IIFE, call stack, recursion · June 2025

What is a Function?

A function is a reusable block of code that you define once and call many times. Instead of repeating the same lines throughout your program, you group them into a function and invoke it wherever needed — saving time, reducing bugs, and making your code maintainable.

● Reusability

Write code once, use it in multiple places. Change it in one place and every call reflects the update.

No copy-pasting the same 10 lines everywhere

● Abstraction

Hide complexity behind a name. Callers don't need to know how a function works — just what it does.

calculateTax(salary) — caller doesn't see the math

● Math Definition

Mathematically, a function maps input to output. You pass values in, get a computed result back.

f(x) = x * 2 → f(5) = 10
Note In JavaScript, functions are first-class citizens — they can be assigned to variables, passed as arguments, and returned from other functions. This is what makes JS so powerful.

Defining & Invoking a Function

Defining (also called declaring) a function creates it but does NOT execute it. Invoking (also called calling) a function actually runs its code.

Syntax — Function Declaration

// DEFINING the function:
function functionName() {
  // function body — code that runs when invoked
}

// INVOKING the function:
functionName(); // ← parentheses trigger execution

Example

// Define
function printThis() {
  console.log("Printing a message");
}
// At this point nothing happens — function exists but hasn't run.

// Invoke
printThis(); // ← now it runs
// Output: "Printing a message"

// Call it again — reusability in action:
printThis();
printThis();
// Output: "Printing a message" (three times total)
Tip Defining a function = like writing a recipe. Invoking it = actually cooking the dish. You can cook the same recipe as many times as you want.

Function Hoisting

Function declarations are fully hoisted. You can call them before they appear in the file.

greet(); // ✅ works! Function declarations are hoisted.

function greet() {
  console.log("Hello!");
}
// Output: "Hello!"

Function Expression

Because functions are first-class citizens in JS, you can assign a function as a value to a variable. This is called a function expression. 

// Function DECLARATION (traditional):
function printThis() {
  console.log("Printing a message");
}

// Function EXPRESSION (assigned to variable):
const printMe = function() {
  console.log("Printing via expression");
};

// printMe is a variable whose value IS a function
console.log(printMe); // prints the function itself

// Invoke it — same as always, just use the variable name + ()
printMe(); // Output: "Printing via expression"
Warning Function expressions are NOT hoisted. You cannot call them before the line where they are defined — doing so throws a ReferenceError or TypeError.

Declaration vs Expression — Key Difference

● Function Declaration

Uses the function keyword as a statement. Fully hoisted — callable anywhere in the file.

function greet() {}

● Function Expression

Function assigned to a variable. Not hoisted — only callable after the line it is defined.

const greet = function() {};

● Anonymous Function

A function with no name. Only valid in expressions or as arguments — no need for a name since you won't call it by name later.

function() { ... }

Parameters & Arguments

Functions become truly powerful when they accept inputs. The inputs let the same function work with different data each time it is called.

● Parameter

The placeholder variable listed in the function definition. Like an empty slot waiting for a value.

Defined at declaration time

● Argument

The actual value passed to the function when it is called. Fills the parameter slot.

Provided at call/invocation time

● Mapping

Arguments map to parameters positionally — first argument → first parameter, second → second, etc.

sum(10, 4) → a=10, b=4
// a and b are PARAMETERS (placeholders in the definition):
function sum(a, b) {
  const result = a + b;
  console.log(result);
}

// 10 and 4 are ARGUMENTS (actual values at the call site):
sum(10, 4);  // Output: 14
sum(5, 7);   // Output: 12 — same function, different data ✅
Note Classic interview question: "What's the difference between a parameter and an argument?" Parameter = variable in the definition. Argument = value passed at the call. Different words, different times.

The return Statement

return does two things: it sends a value back to the caller, and it exits the function immediately. Code after a return never runs. 

function sum(a, b) {
  return a + b;  // sends the result back to whoever called sum()
}

// Capture the returned value in a variable:
const result = sum(10, 9);
console.log(result); // 19

// Chain with another function:
function double(n) {
  return n * 2;
}

console.log(double(result)); // 38 — result(19) * 2

return exits the function immediately

function checkAge(age) {
  if (age < 18) {
    return "Minor";   // exits here if age < 18
  }
  // This line only runs if the return above didn't fire:
  return "Adult";
}

console.log(checkAge(15)); // "Minor"
console.log(checkAge(25)); // "Adult"

⚠️ Critical Bug — return on its own line

JavaScript has Automatic Semicolon Insertion (ASI). If you write the return value on the next line, JS inserts a semicolon after return and your function returns undefined silently.

// ❌ WRONG — JS reads this as: return; (undefined)  then ignores a + b
function sum(a, b) {
  return
    a + b;
}
console.log(sum(2, 3)); // undefined — silent bug!

// ✅ CORRECT — return value must be on the SAME line
function sum(a, b) {
  return a + b;
}
console.log(sum(2, 3)); // 5
Warning Always put the return value on the same line as the return keyword. A newline between return and its value is a silent bug that returns undefined with no error.

Default Parameters

A default parameter provides a fallback value for a parameter when the caller doesn't pass an argument for it. Without defaults, an unset parameter is undefined, which often breaks arithmetic. 

// Without default — missing argument = undefined = NaN
function calc(a, b) {
  return (a + b) * 2;
}
console.log(calc(2));    // NaN — b is undefined, 2 + undefined = NaN

// With default — b falls back to 0 if not provided
function calc(a, b = 0) {
  return (a + b) * 2;
}
console.log(calc(2));    // 4  — b = 0 (default), (2 + 0) * 2 = 4
console.log(calc(2, 7)); // 18 — b = 7 (argument overrides default), (2 + 7) * 2 = 18
// Both parameters with defaults — safe even with no arguments:
function greet(name = "Guest", greeting = "Hello") {
  return `${greeting}, ${name}!`;
}

console.log(greet());                  // "Hello, Guest!"
console.log(greet("Alice"));           // "Hello, Alice!"
console.log(greet("Bob", "Welcome")); // "Welcome, Bob!"
Tip Default parameters make your functions fail-safe. Always provide sensible defaults for optional parameters, especially in utility functions.

Rest Parameter ( ...rest )

The rest parameter (...) lets a function accept an indefinite number of arguments and collects them into an array. Use it when you know some parameters but don't know how many extra values will be passed. 

// x and y are known parameters, ...rest catches everything else
function calculateThis(x, y, ...rest) {
  console.log("x:", x);
  console.log("y:", y);
  console.log("rest:", rest);  // rest is an array
}

calculateThis(1, 2, 3, 4, 5, 6, 7, 8, 9);
// x: 1
// y: 2
// rest: [3, 4, 5, 6, 7, 8, 9]
// Real use case — sum any number of values:
function sumAll(first, ...numbers) {
  let total = first;
  for (let n of numbers) {
    total += n;
  }
  return total;
}

console.log(sumAll(1, 2, 3));          // 6
console.log(sumAll(10, 20, 30, 40));   // 100
Warning The rest parameter must always be the last parameter. Writing function foo(...rest, x) throws a SyntaxError: Rest parameter must be last formal parameter.

Nested Functions

A function defined inside another function is called a nested (or inner) function. The outer function contains it. The inner function is only accessible within the outer function's scope — unless returned. 

function outer() {
  console.log("outer");

  function inner() {
    console.log("inner");
  }

  inner(); // ✅ inner called inside outer — works
}

outer();
// Output: "outer"
//         "inner"

inner(); // ❌ ReferenceError: inner is not defined
// inner doesn't exist outside of outer's scope

Returning the Inner Function

You can make an inner function accessible outside by returning it from the outer function. This is the foundation of closures (covered in a later lecture).

function outer() {
  function inner() {
    console.log("inner called!");
  }
  return inner;  // return the function itself (no parentheses)
}

const retrievedFn = outer();   // outer() runs and returns the inner function
console.log(retrievedFn);      // prints the inner function
retrievedFn();                 // invoke the retrieved inner function
// Output: "inner called!"
Note When you return inner (no parentheses), you're returning the function as a value. When you return inner() (with parentheses), you're returning the result of calling inner. These are completely different.

Callback Function

A callback is a function passed as an argument to another function. The receiving function calls ("calls back") the passed function at some point — often based on a condition or event. 

// foo accepts a function as a parameter (funk)
function foo(funk) {
  console.log("foo");
  funk();  // invoke the passed function
}

// Pass an anonymous function as callback:
foo(function() {
  console.log("buzz");
});
// Output: "foo"
//         "buzz"
// Or pass a named function reference:
function buzz() {
  console.log("buzz");
}

foo(buzz);  // pass buzz as a reference — NO parentheses here!
// Output: "foo"
//         "buzz"
Warning When passing a function as a callback, do NOT include (). Writing foo(buzz()) immediately calls buzz and passes its return value. Writing foo(buzz) passes the function itself. The difference is critical.

Conditional Callback

The callback is often called based on some condition inside the receiving function — not always immediately.

function foo(funk, shouldCall) {
  console.log("foo");
  if (shouldCall) {
    funk();  // only called if shouldCall is true
  }
}

foo(function() { console.log("buzz"); }, true);
// Output: "foo"  "buzz"

foo(function() { console.log("buzz"); }, false);
// Output: "foo"  — buzz is never called
Note Callbacks are the foundation of asynchronous JS — event listeners, setTimeout, API calls. You'll revisit callbacks in depth during the async JavaScript module.

Pure Function

A pure function follows two strict rules: (1) given the same input, it always returns the same output, and (2) it has no side effects — it doesn't modify anything outside itself.

● Same Input → Same Output

Calling the function with the same arguments always produces the same result. No randomness, no time-dependency.

add(2, 3) always returns 5

● No Side Effects

The function does not modify any external variables, objects, or state. It only works with its inputs and returns a new value.

Doesn't mutate global vars, doesn't write to disk

● Predictable

Pure functions are easy to test, debug, and reason about. No hidden state, no surprises.

Unit testing is trivial — no mocking needed
// ✅ PURE — same input always gives same output, no side effects
function add(a, b) {
  return a + b;
}
console.log(add(2, 3)); // always 5
console.log(add(2, 3)); // always 5

// ❌ IMPURE — depends on external variable (side effect)
let multiplier = 2;
function multiply(n) {
  return n * multiplier;  // output depends on external state
}
multiplier = 10;
console.log(multiply(5)); // 50 — changed because multiplier changed!

// ❌ IMPURE — modifies external state
let count = 0;
function increment() {
  count++;  // mutates external variable — side effect
}
increment();
console.log(count); // 1 — external state was changed
Tip Prefer pure functions wherever possible. They make code predictable, testable, and safe to run in parallel. Frameworks like React are built around pure function principles.

Higher Order Function (HOF)

A Higher Order Function is a function that either accepts a function as an argument, or returns a function (or both). This is only possible because JS functions are first-class citizens. 

// HOF — takes a function as argument (callback pattern):
function operate(a, b, operation) {
  return operation(a, b);   // calls the passed function
}

function add(a, b)      { return a + b; }
function subtract(a, b) { return a - b; }
function multiply(a, b) { return a * b; }

console.log(operate(10, 5, add));       // 15
console.log(operate(10, 5, subtract)); // 5
console.log(operate(10, 5, multiply)); // 50
// Same HOF, different behaviour depending on which function you pass in
// HOF — returns a function (function factory):
function multiplier(factor) {
  return function(number) {    // returns a NEW function
    return number * factor;
  };
}

const double = multiplier(2);
const triple = multiplier(3);

console.log(double(5));  // 10
console.log(triple(5));  // 15
Note Built-in HOFs you already use: array.map(), array.filter(), array.reduce(), array.forEach() — they all accept a function as an argument. Now you know why.

Arrow Function ( => )

Introduced in ES6. Arrow functions are a shorter, cleaner syntax for writing function expressions. They also have a different behaviour for the this keyword (covered in a later lecture).

Syntax Evolution

// Traditional function expression:
const add = function(a, b) {
  return a + b;
};

// Arrow function (same thing, cleaner):
const add = (a, b) => {
  return a + b;
};

// Arrow function — implicit return (one-liner, no braces, no return keyword):
const add = (a, b) => a + b;

console.log(add(3, 4)); // 7

Shorthand Rules

  • One parameter — parentheses optional: x => x * 2
  • Zero parameters — parentheses required: () => console.log("hi")
  • Single expression body — omit {} and return, value is implicitly returned
  • Multi-line body — must use {} and explicit return
// One parameter — parens optional:
const square = x => x * x;
console.log(square(5)); // 25

// No parameters:
const greet = () => "Hello, World!";
console.log(greet()); // "Hello, World!"

// Multi-line body — braces + explicit return:
const gradeScore = (score) => {
  if (score >= 90) return "A";
  if (score >= 80) return "B";
  if (score >= 70) return "C";
  return "Fail";
};
console.log(gradeScore(85)); // "B"
// Arrow functions in HOFs (very common pattern):
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

const evens = numbers.filter(n => n % 2 === 0);
console.log(evens);   // [2, 4]
Note Arrow functions do NOT have their own this binding — they inherit this from the enclosing scope. This is intentional and important. Deep-dive in the this keyword lecture.

IIFE — Immediately Invoked Function Expression

An IIFE (pronounced "iffy") is a function that is defined and immediately called at the same time. It runs once — right where it is written — and its internal variables are scoped privately.

Syntax

// Wrap the function in () to make it an expression, then () to invoke:
(function() {
  console.log("IIFE ran!");
})();
// Output: "IIFE ran!" — fires immediately, no need to call it separately
// Arrow function IIFE:
(() => {
  console.log("Arrow IIFE ran!");
})();
// IIFE with parameters:
(function(name) {
  console.log("Hello,", name);
})("Alice");
// Output: "Hello, Alice"

Why Use IIFE?

  • Avoid polluting global scope — variables inside an IIFE are private, invisible outside.
  • One-time setup code — run initialisation logic once without creating a named function.
  • Module pattern — used before ES6 modules to create private scope.
// Without IIFE — counter pollutes global scope:
let counter = 0;
counter++;

// With IIFE — counter is private, global scope stays clean:
(function() {
  let counter = 0;
  counter++;
  console.log("Counter:", counter); // 1
})();

console.log(typeof counter); // "undefined" — counter doesn't exist outside!

The Call Stack

The call stack is how JavaScript tracks which function is currently running and where to return after it finishes. It follows LIFO — Last In, First Out (like a stack of plates).

How It Works

  • When a function is called, it is pushed onto the stack.
  • When the function returns, it is popped off the stack.
  • JS is single-threaded — only one function runs at a time (the one on top of the stack).
function c() {
  console.log("inside c");
}

function b() {
  console.log("inside b — calling c");
  c();
  console.log("back in b");
}

function a() {
  console.log("inside a — calling b");
  b();
  console.log("back in a");
}

a();

// Call Stack progression:
// [a]           ← a is called, pushed
// [a, b]        ← b is called inside a, pushed
// [a, b, c]     ← c is called inside b, pushed
// [a, b]        ← c returns, popped
// [a]           ← b returns, popped
// []            ← a returns, popped — stack empty

// Output:
// "inside a — calling b"
// "inside b — calling c"
// "inside c"
// "back in b"
// "back in a"

Stack Overflow

If functions keep calling each other without ever returning (infinite recursion), the stack fills up completely and crashes with a RangeError: Maximum call stack size exceeded.

// ❌ Infinite recursion — stack overflow!
function infinite() {
  return infinite(); // calls itself forever, never returns
}
infinite(); // RangeError: Maximum call stack size exceeded
Note The call stack is also visible in browser DevTools — the "Sources" tab shows a stack trace whenever you hit a breakpoint or error. Learning to read it is an essential debugging skill.

Recursion

Recursion is when a function calls itself. Each call solves a smaller version of the same problem until it hits a base case — a condition that stops the recursion. Without a base case, you get infinite recursion and a stack overflow.

Structure of a Recursive Function

function recursive(input) {
  // 1. BASE CASE — stop condition (MUST have this!)
  if (input === baseCondition) {
    return someValue;
  }

  // 2. RECURSIVE CASE — call itself with a smaller problem
  return recursive(smallerInput);
}

Example — Countdown

function countdown(n) {
  if (n <= 0) {        // base case — stop at 0
    console.log("Go!");
    return;
  }
  console.log(n);
  countdown(n - 1);   // recursive call — smaller problem each time
}

countdown(5);
// Output: 5  4  3  2  1  "Go!"

Classic Example — Factorial

Factorial of n (written n!) = n × (n-1) × (n-2) × ... × 1. Example: 5! = 5 × 4 × 3 × 2 × 1 = 120.

function factorial(n) {
  if (n === 1 || n === 0) return 1;  // base case: 1! = 1, 0! = 1
  return n * factorial(n - 1);       // recursive case
}

// How it resolves for factorial(5):
// factorial(5) = 5 * factorial(4)
// factorial(4) = 4 * factorial(3)
// factorial(3) = 3 * factorial(2)
// factorial(2) = 2 * factorial(1)
// factorial(1) = 1  ← base case hit, unwind begins
// ← 2 * 1 = 2
// ← 3 * 2 = 6
// ← 4 * 6 = 24
// ← 5 * 24 = 120

console.log(factorial(5)); // 120
console.log(factorial(6)); // 720

Recursion vs Loops

● Loops

Better for simple repetition. Efficient — no stack overhead. Easier to read for basic iteration.

Use for: counting, summing, iterating arrays

● Recursion

Better for problems that have naturally recursive structure — trees, graphs, nested data, divide-and-conquer.

Use for: file trees, DOM traversal, sorting algorithms

● Always have a Base Case

Every recursive function MUST have a condition that stops it. Missing base case = stack overflow crash.

if (n === 0) return;
Tip Think of recursion as: "Solve the smallest version of this problem first (base case), then build up from there." If you can't define the base case, you can't write the recursion.

Key Takeaways

  • A function groups reusable code. Define once, invoke many times.
  • Declaration: function name() {} — hoisted, callable before definition. Expression: const f = function() {} — not hoisted.
  • Functions are first-class citizens — assignable to variables, passable as arguments, returnable from functions.
  • Parameter = placeholder in definition. Argument = actual value at call site.
  • return sends a value back and exits the function. Return value must be on the same line as return.
  • Default parameter: function f(a, b = 0) — fallback when argument is not provided.
  • Rest parameter: ...rest — collects extra arguments into an array. Must be the last parameter.
  • Nested function — inner function only accessible inside outer, unless returned.
  • Callback — function passed as argument, invoked later (possibly conditionally). Pass by reference, not by calling.
  • Pure function — same input always = same output, no side effects. Prefer these.
  • HOF (Higher Order Function) — accepts or returns a function. map, filter, reduce are HOFs.
  • Arrow function: (a, b) => a + b — concise syntax, implicit return on one-liners, no own this.
  • IIFE: (function(){})(); — runs immediately, keeps variables out of global scope.
  • Call stack — LIFO structure tracking function execution. Full stack = RangeError: Maximum call stack size exceeded.
  • Recursion — function calls itself. Needs a base case. Good for tree/nested problems. Missing base case = stack overflow.
