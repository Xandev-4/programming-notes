Operators & Expressions in JavaScript

Operands, operators, expressions, all operator types, precedence, typeof & instanceof · June 2025

Operands, Operators & Expressions

Before diving into specific operators, you need to nail the three core terms. These are the building blocks of all logic in JavaScript.

● Operator

A symbol that performs an operation on values. Like a verb — it does something.

e.g. + - * / === &&

● Operand

The values the operator acts on. Like nouns — they are what gets operated on.

e.g. in x + y, both x and y are operands

● Expression

Any combination of operands and operators that resolves to a single value.

e.g. 4 + 5 resolves to 9

Two Types of Expressions

  • Assignment Expression — assigns a value to a variable: let x = 10;
  • Evaluating Expression — combines operands and operators to produce a new value: 3 + 4 → 7
// Both types combined into one expression:
let x = 4 + 5;
// Here: 4 + 5 is an evaluating expression (resolves to 9)
// let x = 9  is the assignment expression
console.log(x); // 9
Note An expression ALWAYS resolves to a value. If it doesn't produce a value, it's a statement, not an expression.

Arithmetic Operators

Arithmetic operators take numeric values as operands and return a single numeric value. These are the most intuitive operators.

Basic Arithmetic

let a = 10;
let b = 20;

console.log(a + b);  // 30  — Addition
console.log(a - b);  // -10 — Subtraction
console.log(b - a);  // 10
console.log(a * b);  // 200 — Multiplication
console.log(a / b);  // 0.5 — Division

String Concatenation with +

When + is used with strings, it concatenates them instead of adding. This is a common source of bugs.

let firstName = "Tapas";
let lastName = "Adhikari";

console.log(firstName + lastName); // "TapasAdhikari" — no space!

// ⚠️ Avoid using + for string joining.
// Use template literals instead (covered in a future lecture):
console.log(`${firstName} ${lastName}`); // "Tapas Adhikari"
Warning If one operand is a string and you use +, JS coerces the other operand to a string too. "5" + 3 gives "53", NOT 8. Use template literals for string building.

Exponentiation Operator ( ** )

Raises the left operand to the power of the right operand. Works like Math.pow().

let a = 2;
let b = 3;
console.log(a ** b); // 8 — (2 to the power of 3 = 2 × 2 × 2)

console.log(10 ** 20); // 100000000000000000000 (10 to the power 20)

Remainder / Modulo Operator ( % )

Returns the remainder left over after integer division. Not a percentage sign here.

let a = 12;
let b = 5;

// 12 ÷ 5 = 2 remainder 2
console.log(a % b); // 2

// Common use: check if a number is even or odd
console.log(10 % 2); // 0 — even (no remainder)
console.log(11 % 2); // 1 — odd

Increment Operators ( ++ )

Adds 1 to a variable. BUT the position of ++ (before or after the variable) changes the behaviour critically.

let count = 5;

// POST-INCREMENT (count++) — return value FIRST, then increment
console.log(count++); // 5  ← returns 5, THEN increments to 6
console.log(count);   // 6  ← proof: count is now 6

// PRE-INCREMENT (++count) — increment FIRST, then return value
let count2 = 5;
console.log(++count2); // 6  ← increments to 6 FIRST, then returns 6
console.log(count2);   // 6
Tip Post-increment (count++) = "use now, update later." Pre-increment (++count) = "update now, use updated value." This is a classic interview question.

Decrement Operators ( -- )

Subtracts 1 from a variable. Same post/pre rules apply.

let count = 5;

console.log(count--); // 5  ← returns 5, then decrements to 4
console.log(count);   // 4

let count2 = 5;
console.log(--count2); // 4  ← decrements to 4 first, then returns 4

Assignment Operators

Assignment operators assign a value to a variable. The basic one is = (which we've already used). The rest are shorthand compound operators that combine an operation with assignment.

let x = 10; // basic assignment

// x += 5  is shorthand for  x = x + 5
x += 5;
console.log(x); // 15

// x -= 3  is shorthand for  x = x - 3
x -= 3;
console.log(x); // 12

// x *= 2  is shorthand for  x = x * 2
x *= 2;
console.log(x); // 24

// x /= 4  is shorthand for  x = x / 4
x /= 4;
console.log(x); // 6

// x **= 2  is shorthand for  x = x ** 2
x **= 2;
console.log(x); // 36

// x %= 5  is shorthand for  x = x % 5
x %= 5;
console.log(x); // 1
Note Each compound assignment (+=, -=, etc.) always operates on the current value of the variable at that line — not the original declared value. Track the running value carefully.

Comparison Operators

Comparison operators compare two values and always return a Boolean (true or false). This is what drives all your if conditions and logic.

Loose Equality ( == ) — AVOID

Checks if two values are equal, but automatically converts types (type coercion) before comparing. This causes unexpected results.

console.log(0 == false);   // true  ← WEIRD! 0 is a number, false is boolean
console.log("3" == 3);     // true  ← WEIRD! string "3" vs number 3
console.log(null == undefined); // true

Why? JavaScript converts both operands to the nearest compatible type and then compares. Zero and false are both "falsy" so they are treated as equal. This is unpredictable.

Strict Equality ( === ) — ALWAYS USE THIS

Checks both value AND type. No type coercion. If types differ, returns false immediately.

console.log(0 === false);  // false ← correct! different types
console.log("3" === 3);    // false ← correct! different types
console.log(3 === 3);      // true  ← same value, same type ✅

// Rules for strict equality:
console.log(null === null);           // true
console.log(undefined === undefined); // true
console.log(NaN === NaN);             // false ← NaN is NEVER equal to anything, even itself!
Warning Object comparison with === compares references (memory addresses), NOT the content. Two objects with identical values are NOT strictly equal unless they point to the same memory location. 
let obj1 = { name: "Alice" };
let obj2 = { name: "Alice" };
let obj3 = obj1;  // obj3 points to the SAME memory as obj1

console.log(obj1 === obj2); // false — different memory addresses
console.log(obj1 === obj3); // true  — same memory address ✅

Loose Not-Equal ( != ) — AVOID

console.log(0 != false);  // false ← misleading, same coercion problem

Strict Not-Equal ( !== ) — ALWAYS USE THIS

console.log(0 !== false);  // true  ← correct, they are different types
console.log(3 !== 4);      // true  ← 3 is not 4

Greater/Less Than Operators

console.log(4 > 3);   // true  — greater than
console.log(1 > 7);   // false

console.log(4 >= 4);  // true  — greater than OR equal to (either condition is enough)
console.log(2 >= 4);  // false

console.log(4 < 3);   // false — less than
console.log(2 < 7);   // true

console.log(3 <= 9);  // true  — less than OR equal to
console.log(3 <= 3);  // true  — equal satisfies it
Note >= checks "greater than OR equal to" — only one condition needs to be true. Common mistake: writing => which is an arrow function, not a comparison operator!

Logical Operators

Logical operators combine or invert Boolean conditions. There are four: &&, ||, ??, and !. Understanding their return rules (not just true/false) is critical.

Logical AND ( && )

Rule: If the first operand can be converted to false, return the first operand. Otherwise, return the second operand.

console.log(false && false); // false — first is false, return first
console.log(true  && false); // false — first is true, return second (false)
console.log(true  && true);  // true  — first is true, return second (true)
console.log(false && true);  // false — first is false, return first

// With non-booleans:
console.log("cow" && "horse"); // "horse" — "cow" is truthy, so return second operand
console.log(0 && "horse");     // 0      — 0 is falsy, so return first operand

Shortcut memory: && returns false if ANY operand is falsy. Only returns truthy when BOTH sides are truthy.

// Real-world logic:
console.log(4 > 5 && 4 === 6); // false && false → false
console.log(5 > 4 && 6 === 6); // true && true  → true

Logical OR ( || )

Rule: If the first operand can be converted to true, return the first operand. Otherwise, return the second operand.

console.log(false || false); // false — first is false, return second (false)
console.log(true  || false); // true  — first is true, return first (true)
console.log(true  || true);  // true  — first is true, return first (true)
console.log(false || true);  // true  — first is false, return second (true)

// With non-booleans:
console.log("cow" || "horse"); // "cow"   — "cow" is truthy, return first
console.log(0 || "horse");     // "horse" — 0 is falsy, return second

// Real-world use: default value fallback
let username = null;
let displayName = username || "Guest";
console.log(displayName); // "Guest" — username is falsy, fallback to "Guest"
Tip Shortcut: || returns true if ANY operand is truthy. Both must be falsy to return falsy. It's the opposite logic of &&.

Nullish Coalescing Operator ( ?? )

Like ||, but only treats null and undefined as falsy. It does NOT treat 0, "", or false as fallback triggers. This is a safer default-value operator.

let score = 0;

// Problem with ||: 0 is falsy, so it falls back to 100 — WRONG!
console.log(score || 100);  // 100 ← incorrect, we wanted 0

// Fix with ??: only null/undefined trigger the fallback
console.log(score ?? 100);  // 0   ← correct! 0 is a valid value

let name = null;
console.log(name ?? "Anonymous"); // "Anonymous" — null triggers fallback ✅

let age = undefined;
console.log(age ?? 18); // 18 — undefined triggers fallback ✅
Tip Use ?? when 0, false, or "" are valid values you want to keep. Use || when any falsy value should trigger the fallback.

Logical NOT ( ! )

Inverts a Boolean value. true becomes false, false becomes true.

console.log(!true);   // false
console.log(!false);  // true

// Double NOT (!!) converts any value to its Boolean equivalent
console.log(!!0);        // false — 0 is falsy
console.log(!!"hello");  // true  — non-empty string is truthy
console.log(!!null);     // false — null is falsy
console.log(!!1);        // true  — non-zero number is truthy

Conditional (Ternary) Operator

The only JavaScript operator that takes three operands. It's a compact one-liner version of an if...else statement.

Syntax

condition ? valueIfTrue : valueIfFalse

Examples

let age = 20;
let access = age >= 18 ? "Allowed" : "Denied";
console.log(access); // "Allowed"

let score = 45;
let result = score >= 50 ? "Pass" : "Fail";
console.log(result); // "Fail"

// Equivalent if...else for reference:
// if (score >= 50) {
//   result = "Pass";
// } else {
//   result = "Fail";
// }
Tip Use the ternary operator for simple, single-condition checks. If the logic gets complex (nested ternary), switch to a regular if...else for readability.

Bitwise Operators

Bitwise operators work on the binary (bit-level) representation of numbers. They treat operands as 32-bit integers and operate on individual bits. Rarely used in everyday JS, but important to know.

● AND ( & )

Returns 1 for each bit position where BOTH operands have 1.

5 & 3 → 1

● OR ( | )

Returns 1 for each bit position where AT LEAST ONE operand has 1.

5 | 3 → 7

● XOR ( ^ )

Returns 1 for each bit position where EXACTLY ONE operand has 1 (not both).

5 ^ 3 → 6
// How it works under the hood:
// 5 in binary = 0101
// 3 in binary = 0011

console.log(5 & 3);  // 1  → 0101 AND 0011 = 0001 (only bit 0 is 1 in both)
console.log(5 | 3);  // 7  → 0101 OR  0011 = 0111
console.log(5 ^ 3);  // 6  → 0101 XOR 0011 = 0110

// NOT (~) — inverts all bits
console.log(~5);     // -6  (flips all 32 bits, result is -(n+1))

// Left Shift (<<) — shifts bits left, fills with 0s (multiplies by powers of 2)
console.log(5 << 1); // 10  → 0101 becomes 1010

// Right Shift (>>) — shifts bits right (divides by powers of 2, keeps sign)
console.log(5 >> 1); // 2   → 0101 becomes 0010

// Zero-fill Right Shift (>>>) — shifts right, fills with 0s (unsigned)
console.log(5 >>> 1); // 2
Note Bitwise operators are mostly used in low-level operations, encryption, flags, and performance-critical code. You won't need them often in standard web dev, but they appear in interviews and certain algorithms.

Relational Operators

Relational operators check the relationship between two entities. Two main ones: in and instanceof.

in Operator

Checks whether a property exists in an object (or an index in an array). Returns a Boolean.

let car = { make: "Toyota", model: "Camry", year: 2022 };

console.log("make" in car);    // true  — "make" is a key in car
console.log("color" in car);   // false — "color" is not a key in car

let fruits = ["apple", "mango", "kiwi"];
console.log(0 in fruits);  // true  — index 0 exists
console.log(5 in fruits);  // false — index 5 does not exist

Grouping & Operator Precedence

When multiple operators appear in one expression, JavaScript uses operator precedence to decide which operation to perform first — just like BODMAS/PEMDAS in maths. You can override precedence using parentheses ().

Precedence Problem

// Without grouping — follows precedence (* before +)
let result = 2 + 3 * 4;
console.log(result); // 14 — (3 * 4 = 12, then 2 + 12 = 14)

// With grouping — parentheses override precedence
let result2 = (2 + 3) * 4;
console.log(result2); // 20 — (2 + 3 = 5, then 5 * 4 = 20)

Precedence Order (High to Low — simplified)

  • Grouping () — highest priority, always runs first
  • Unary operators — !, ++, --, typeof, - (negation)
  • Exponentiation **
  • Multiplication, Division, Remainder — *, /, %
  • Addition, Subtraction — +, -
  • Comparison — >, <, >=, <=
  • Equality — ===, !==, ==, !=
  • Logical AND — &&
  • Logical OR — ||
  • Nullish Coalescing — ??
  • Ternary — ? :
  • Assignment — =, +=, etc. — lowest priority
// Complex example — can you predict the output?
let result = 10 + 2 * 3 - 4 / 2;
// Step 1: 2 * 3 = 6
// Step 2: 4 / 2 = 2
// Step 3: 10 + 6 - 2 = 14
console.log(result); // 14

// With grouping to change the outcome:
let result2 = (10 + 2) * (3 - 4) / 2;
// Step 1: (10 + 2) = 12
// Step 2: (3 - 4) = -1
// Step 3: 12 * -1 = -12
// Step 4: -12 / 2 = -6
console.log(result2); // -6
Tip When in doubt, use parentheses. They make your intent explicit and prevent precedence-related bugs. Code is for humans first, machines second.

typeof Operator

The typeof operator returns a string indicating the type of a value. It's unary — takes one operand. Extremely useful for debugging and type-checking at runtime.

console.log(typeof "hello");       // "string"
console.log(typeof 42);            // "number"
console.log(typeof 3.14);          // "number"
console.log(typeof true);          // "boolean"
console.log(typeof undefined);     // "undefined"
console.log(typeof null);          // "object"  ← Famous JS bug! null is NOT an object
console.log(typeof Symbol("id"));  // "symbol"
console.log(typeof 9007199n);      // "bigint"
console.log(typeof function(){}); // "function"
console.log(typeof {});            // "object"
console.log(typeof []);            // "object"  ← arrays are objects in JS
Warning typeof null returns "object" — this is a historic bug in JS that can never be fixed (too much code depends on it). To check for null, always use value === null.

Practical Use

function processInput(input) {
  if (typeof input === "string") {
    console.log("Got a string:", input.toUpperCase());
  } else if (typeof input === "number") {
    console.log("Got a number:", input * 2);
  } else {
    console.log("Unknown type:", typeof input);
  }
}

processInput("hello"); // "Got a string: HELLO"
processInput(5);       // "Got a number: 10"
processInput(true);    // "Unknown type: boolean"

instanceof Operator

Checks whether an object was created by a specific constructor or class. Returns a Boolean. Unlike typeof, it works with complex types and class hierarchies.

// instanceof with built-in types
let arr = [1, 2, 3];
let obj = { name: "Alice" };
let today = new Date();

console.log(arr instanceof Array);   // true
console.log(arr instanceof Object);  // true  ← arrays are also objects
console.log(obj instanceof Object);  // true
console.log(today instanceof Date);  // true
console.log(arr instanceof Date);    // false

typeof vs instanceof

● typeof

Returns a string for the primitive type. Works for primitives. Unreliable for objects and arrays (all return "object").

Use for: strings, numbers, booleans, undefined

● instanceof

Returns a boolean. Works for objects — tells you exactly which class/constructor created it.

Use for: Array, Date, custom classes

● === null

The only reliable way to check for null since typeof null returns "object".

Use: value === null
// The right tool for each check:
let value = [1, 2, 3];

console.log(typeof value);             // "object"      ← not helpful for arrays
console.log(value instanceof Array);   // true          ← precise ✅
console.log(Array.isArray(value));     // true          ← most reliable for arrays

Falsy & Truthy Values

In JavaScript, every value is either truthy or falsy. This matters because logical operators, if conditions, and ternary operators all work based on this truthiness — not just strict true/false.

All Falsy Values (only 7 in total)

// These are the ONLY falsy values in JavaScript:
false
0
-0
0n          // BigInt zero
""          // empty string (single or double quotes)
null
undefined
NaN

// Everything else is truthy — including:
"0"         // non-empty string (even if it contains "0") → TRUTHY
[]          // empty array → TRUTHY
{}          // empty object → TRUTHY
-1          // any non-zero number → TRUTHY
// Practical impact on logical operators:
console.log(0 || "fallback");      // "fallback" — 0 is falsy
console.log("" || "default");      // "default"  — empty string is falsy
console.log([] || "never");        // []         — empty array is TRUTHY!
console.log(null ?? "safe");       // "safe"     — null triggers ??

// In conditions:
if (0) { console.log("runs"); }         // Does NOT run — 0 is falsy
if ("hello") { console.log("runs"); }   // Runs — non-empty string is truthy
Tip Memorise the 7 falsy values. Everything else is truthy — including [], {}, "false", and -1. This trips up beginners constantly.

Key Takeaways

  • Operator = symbol that performs operation. Operand = value it operates on. Expression = resolves to a value.
  • Arithmetic: + - * / % **. + with strings = concatenation (use template literals instead).
  • Post-increment count++ = return first, then increment. Pre-increment ++count = increment first, then return.
  • Compound assignment: x += 5 is shorthand for x = x + 5. Always uses current value of variable.
  • == does type coercion — avoid it. === checks value AND type — always use this.
  • Object equality with === compares memory references, not content.
  • &&: returns first falsy operand or last truthy one. ||: returns first truthy operand or last falsy one.
  • ??: only null/undefined trigger fallback — safer than || when 0 or "" are valid values.
  • Ternary condition ? a : b is a compact if/else that resolves to a value.
  • Bitwise operators work on binary representations of numbers. Rarely used in standard web dev.
  • typeof returns a string of the type. typeof null === "object" is a famous JS bug.
  • instanceof checks which constructor/class created an object.
  • Parentheses () have the highest precedence — use them to make your intent explicit.
  • Only 7 falsy values: false, 0, -0, 0n, "", null, undefined, NaN. Everything else is truthy.
