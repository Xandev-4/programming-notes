Building Beginner JavaScript Projects

Project thinking, assumptions, logic building, random numbers, browser APIs — applied through Rock Paper Scissors & Number Guessing Game · June 2025

How to Approach a Project

The biggest beginner mistake is jumping straight into code. Before writing a single line, you must think — understand the problem, break it into pieces, and note your assumptions. This thinking phase is what separates good developers from great ones.

The Correct Order

  1. Understand the requirements — What exactly does this program need to do? What are the rules?
  2. List your assumptions — Before coding, write down what you assume to be true. These may change as you code, and that's fine.
  3. Basic code structure — Scaffold the function(s) and overall layout without logic yet.
  4. Build the logic step by step — Fill in the logic. Test small pieces as you go, not all at the end.
  5. Test iteratively — After each logical step, test it. Don't wait until the end to find bugs.
Tip Listing assumptions is called pre-coding analysis. It's normal for assumptions to change mid-build. The point is to start with a clear mental model so you're not guessing what to code next.

Browser APIs — prompt() & alert()

JavaScript running in a browser has access to the global window object, which provides built-in methods to interact with users. Two critical ones for these projects are prompt() and alert(). These are browser-only — they do not work in Node.js.

● window.alert()

Shows a popup dialog with a message. User can only click OK. Does not return a value.

Use: display info, announce results

● window.prompt()

Shows a popup with a text input. User types something and clicks OK. Returns the input as a string. Returns null if user clicks Cancel.

Use: collect text input from user

● window.confirm()

Shows a dialog with OK/Cancel. Returns true for OK, false for Cancel. Good for yes/no questions.

Use: confirm before deleting, ask to continue
// alert — show a message
window.alert("Welcome to the game!");
// or just:
alert("Welcome to the game!");  // window. prefix is optional

// prompt — collect input from user
const userInput = prompt("Enter rock, paper, or scissors:");
console.log(userInput); // whatever the user typed, as a STRING
// If user clicks Cancel → userInput is null

// The prompt dialog has two buttons:
// OK     → returns the typed string (even empty string "" if nothing typed)
// Cancel → returns null
Warning prompt() always returns a string or null. If you need a number, convert it: Number(prompt("Enter a number:")). Always handle the null case (Cancel button).

Handling the Cancel (null) Case

const rawInput = prompt("Enter yes or no:");

// rawInput is null if user clicked Cancel
// Use nullish coalescing to provide a safe fallback:
const userChoice = rawInput ? rawInput.toLowerCase() : "no";

// Or with optional chaining:
const userChoice = rawInput?.toLowerCase() ?? "no";

console.log(userChoice); // "no" if cancelled, otherwise lowercased input

Random Number Generation

Math.random() is the foundation of any randomness in JavaScript. It returns a decimal (float) between 0 (inclusive) and 1 (exclusive) — i.e., 0.0 to 0.999.... To get a usable integer in a range, you must scale and floor it.

The Formula — Integer Between 1 and N

// Step 1: Math.random() gives a decimal 0.0 – 0.999...
// Step 2: Multiply by N (upper limit) to scale it to 0.0 – N.999...
// Step 3: Math.floor() strips the decimal → integer 0 to (N-1)
// Step 4: Add 1 to shift range from 0–(N-1) to 1–N

const randomNumber = Math.floor(Math.random() * N) + 1;

// Examples:
const oneToThree  = Math.floor(Math.random() * 3)  + 1; // 1, 2, or 3
const oneToTen    = Math.floor(Math.random() * 10) + 1; // 1 to 10
const oneToHundred = Math.floor(Math.random() * 100) + 1; // 1 to 100

Why This Formula Works

// Say Math.random() gives 0.73
// For N = 3:
// 0.73 * 3    = 2.19
// Math.floor(2.19) = 2
// 2 + 1       = 3  ← final result: 3

// Say Math.random() gives 0.05
// 0.05 * 3    = 0.15
// Math.floor(0.15) = 0
// 0 + 1       = 1  ← final result: 1

// Say Math.random() gives 0.99
// 0.99 * 3    = 2.97
// Math.floor(2.97) = 2
// 2 + 1       = 3  ← final result: 3
Note Without the + 1, the range would be 0 to N-1. The +1 shifts it to 1 to N. Change N to change the upper bound. The lower bound is always 1 with this formula.

Project 1 — Rock Paper Scissors

A two-player game between the human and the computer. The human picks via prompt, the computer picks randomly. A winner is announced each round based on the classic rules.

Game Rules

● Rock beats Scissors

A rock can break scissors. If one picks rock and the other picks scissors, rock wins.

● Paper beats Rock

Paper can wrap a rock. If one picks paper and the other picks rock, paper wins.

● Scissors beats Paper

Scissors can cut paper. If one picks scissors and the other picks paper, scissors wins.

Project Assumptions (pre-coding checklist)

  • Use prompt() to collect user's choice.
  • Computer's choice is generated randomly from {rock, paper, scissors}.
  • Compare user's and computer's choices against all winning conditions.
  • Handle a tie (both pick the same thing).
  • Handle invalid input (user types something unexpected).
  • (Optional) Ask the user to play again after each round.

Tip Always .toLowerCase() user input before comparing. "Rock", "ROCK", "rock" are all different strings in JS. Normalising to lowercase means you only need to compare against "rock".

Step 4 — Generate Computer's Choice

function rockPaperScissorsGame() {
  const userChoicePrompt = prompt("Enter rock, paper, or scissors:");
  const userChoice = userChoicePrompt ? userChoicePrompt.toLowerCase() : "";

  // Generate random number 1, 2, or 3
  const randomNumber = Math.floor(Math.random() * 3) + 1;

  // Map random number to a choice
  let computerChoice;
  if      (randomNumber === 1) computerChoice = "rock";
  else if (randomNumber === 2) computerChoice = "paper";
  else                         computerChoice = "scissors";

  console.log("User selected:", userChoice);
  console.log("Computer selected:", computerChoice);
}
Note computerChoice uses let (not const) because it is reassigned across the if-else branches. randomNumber uses const because it is set once and never changed.

Step 5 — Determine the Winner

// The 3 conditions where USER wins:
// rock vs scissors, paper vs rock, scissors vs paper
if (
  (userChoice === "rock"     && computerChoice === "scissors") ||
  (userChoice === "paper"    && computerChoice === "rock")     ||
  (userChoice === "scissors" && computerChoice === "paper")
) {
  console.log("🎉 You win! Yay!");
}
// Tie — both picked the same
else if (userChoice === computerChoice) {
  console.log("🤝 It's a tie! The game is a draw.");
}
// The 3 conditions where COMPUTER wins (mirror of user wins):
else if (
  (userChoice === "rock"     && computerChoice === "paper")    ||
  (userChoice === "paper"    && computerChoice === "scissors") ||
  (userChoice === "scissors" && computerChoice === "rock")
) {
  console.log("💻 Computer wins! Better luck next time.");
}
// Invalid input — none of the above matched
else {
  console.log("⚠️ Could not determine a winner. Please check your input.");
}

Step 6 — Play Again? (using recursion)

// After announcing winner, ask if user wants to play again
const playAgainPrompt = prompt("Do you want to play again? (yes / no)");

// Handle null (Cancel) → treat as "no"
const playAgain = playAgainPrompt ? playAgainPrompt.toLowerCase() : "no";

if (playAgain === "yes") {
  rockPaperScissorsGame();  // recursive call — start the game again!
} else {
  console.log("Thanks for playing! See you next time.");
}
Note Calling the game function from inside itself is recursion! The base case here is when playAgain !== "yes" — that's when it stops calling itself. Each new game gets its own fresh call stack frame.

Project 2 — Number Guessing Game

A solo game where the computer picks a secret number between 1 and 10. The user guesses repeatedly; the game tells them if the guess is too high or too low, tracks the number of attempts, and announces when they get it right.

Game Rules & Expected Behaviour

  • Computer generates a secret number between 1 and 10.
  • User is prompted to guess the number.
  • If the guess is too high → tell the user "Too high, try again."
  • If the guess is too low → tell the user "Too low, try again."
  • If the guess is correct → announce win + how many attempts it took.
  • After winning, ask if the user wants to play again.

Project Assumptions (pre-coding checklist)

  • Declare maxNumber = 10 and minNumber = 1 for the range.
  • Secret number = random integer between minNumber and maxNumber.
  • Use a while loop — we don't know how many guesses it will take (unknown iterations).
  • Track attempts counter — increment each time a guess is made.
  • Convert prompt string to number before comparing (prompt always returns a string).
  • Handle the play-again flow with recursion (same pattern as project 1).

Step 2 — The Guessing Loop

We use a while loop because we don't know in advance how many guesses the user will need. The loop keeps running until the correct number is guessed.

function numberGuessingGame() {
  const minNumber = 1;
  const maxNumber = 10;
  const secretNumber = Math.floor(Math.random() * maxNumber) + minNumber;

  let attempts = 0;      // track how many guesses user has made
  let hasWon = false;    // flag — becomes true when correct guess is made

  while (!hasWon) {
    // Prompt user for a guess
    const guessInput = prompt(`Guess a number between ${minNumber} and ${maxNumber}:`);

    // Handle Cancel (null input) → exit game
    if (guessInput === null) {
      console.log("Game cancelled. See you next time!");
      return;  // exits the function entirely
    }

    // Convert string to number
    const guess = Number(guessInput);

    // Handle NaN (user typed something non-numeric)
    if (isNaN(guess)) {
      console.log("⚠️ That's not a valid number. Try again.");
      continue;  // skip rest of loop, ask again
    }

    attempts++;  // valid guess — count it

    // ── Compare guess to secret ──
    if (guess < secretNumber) {
      console.log(`📉 Too low! (${guess} < secret) Try a higher number.`);
    } else if (guess > secretNumber) {
      console.log(`📈 Too high! (${guess} > secret) Try a lower number.`);
    } else {
      // Correct!
      console.log(`🎉 Correct! The secret number was ${secretNumber}.`);
      console.log(`You guessed it in ${attempts} attempt${attempts > 1 ? "s" : ""}!`);
      hasWon = true;  // exit the while loop
    }
  }
}

  // After the while loop ends (hasWon is true):

  if (playAgain === "yes") {

Complete Number Guessing Game Code

function numberGuessingGame() {
  // ── 1. Setup ───────────────────────────────────────────
  const minNumber = 1;
  const maxNumber = 10;
  const secretNumber = Math.floor(Math.random() * maxNumber) + minNumber;

  let attempts = 0;
  let hasWon = false;

  console.log(`Welcome to the Number Guessing Game!`);
  console.log(`Try to guess the secret number between ${minNumber} and ${maxNumber}.`);

  // ── 2. Guessing loop ───────────────────────────────────
  while (!hasWon) {
    const guessInput = prompt(`Guess a number between ${minNumber} and ${maxNumber}:`);

    // User clicked Cancel
    if (guessInput === null) {
      console.log("Game cancelled. See you next time!");
      return;
    }

    const guess = Number(guessInput);

    // User typed something non-numeric
    if (isNaN(guess)) {
      console.log("⚠️ Please enter a valid number.");
      continue;
    }

    attempts++;

    if (guess < secretNumber) {
      console.log(`📉 Too low! Try a higher number.`);
    } else if (guess > secretNumber) {
      console.log(`📈 Too high! Try a lower number.`);
    } else {
      console.log(`🎉 Yes! The secret number was ${secretNumber}.`);
      console.log(`You guessed it in ${attempts} attempt${attempts > 1 ? "s" : ""}!`);
      hasWon = true;
    }
  }

  // ── 3. Play again? ─────────────────────────────────────
  const playAgainPrompt = prompt("Do you want to play again? (yes / no)");
  const playAgain = playAgainPrompt ? playAgainPrompt.toLowerCase() : "no";

  if (playAgain === "yes") {
    numberGuessingGame();
  } else {
    console.log("Thanks for playing! Goodbye. 👋");
  }
}

// Start the game:
numberGuessingGame();

Concepts From Lec 1–6 Applied in These Projects

These two projects are a deliberate recap of everything covered so far. Here's a direct map of which concept was applied where.

● Variables (Lec 2)

const for fixed values (secretNumber, randomNumber), let for values that change (computerChoice, attempts, hasWon).

● Operators (Lec 3)

&& to combine win conditions, === for strict comparisons, % for even/odd, ternary for pluralising "attempt".

● Control Flow (Lec 4)

if-else if-else chains for winner determination and high/low feedback. Each branch is mutually exclusive.

● Loops (Lec 5)

while (!hasWon) loop — unknown iteration count, runs until condition changes. continue to skip invalid input. return to exit on Cancel.

● Functions (Lec 6)

All logic wrapped in a named function. Recursion for "play again" — function calls itself. return for early exit on Cancel.

● String Methods

.toLowerCase() on user input to normalise casing before comparison. Template literals for dynamic messages.

Tip Every time you build a project, map which concepts you're using. If you find yourself unsure about a concept, that's the one to revisit in the earlier lectures — not after the whole project is done.

Common Bugs in These Projects & How to Fix Them

Bug 1 — prompt() returns a string, not a number

// ❌ Bug: comparing string to number
const guess = prompt("Enter a number:"); // returns "5" (string)
if (guess === 5) { ... }  // false! "5" !== 5

// ✅ Fix: convert to number first
const guess = Number(prompt("Enter a number:"));
if (guess === 5) { ... }  // true ✅

Bug 2 — Not handling null (Cancel button)

// ❌ Bug: calling .toLowerCase() on null crashes the app
const input = prompt("Enter choice:"); // user clicks Cancel → null
const choice = input.toLowerCase();    // TypeError: Cannot read properties of null

// ✅ Fix 1: ternary guard
const choice = input ? input.toLowerCase() : "";

// ✅ Fix 2: optional chaining + nullish coalescing
const choice = input?.toLowerCase() ?? "";

Bug 3 — Using const when value will be reassigned

// ❌ Bug: computerChoice must change across branches
const computerChoice;          // SyntaxError: missing initializer in const declaration
if (random === 1) computerChoice = "rock"; // TypeError: Assignment to constant variable

// ✅ Fix: use let
let computerChoice;
if (random === 1) computerChoice = "rock";

Bug 4 — Comparison is case-sensitive

// ❌ Bug: "Rock" !== "rock"
const userChoice = prompt("Enter choice:"); // user types "Rock"
if (userChoice === "rock") { ... } // false!

// ✅ Fix: normalise casing on input
const userChoice = prompt("Enter choice:").toLowerCase(); // "rock" ✅

Bug 5 — Infinite loop (no exit condition)

// ❌ Bug: hasWon never becomes true
let hasWon = false;
while (!hasWon) {
  const guess = Number(prompt("Guess:"));
  if (guess === secretNumber) {
    console.log("Correct!");
    // forgot to set hasWon = true → loop never ends!
  }
}

// ✅ Fix: update the condition variable
if (guess === secretNumber) {
  console.log("Correct!");
  hasWon = true; // ← this exits the while loop ✅
}

Key Takeaways

  • Don't rush to code. Always understand the requirements and list assumptions first.
  • Test iteratively — after each logical step, test it. Don't wait until the end.
  • prompt() collects user input as a string. Returns null if user clicks Cancel. Always handle both cases.
  • alert() shows a popup. confirm() returns true/false. Both are browser-only window methods.
  • Random integer formula: Math.floor(Math.random() * N) + 1 gives 1 to N.
  • Use const for values set once (secretNumber, randomNumber). Use let for values that change (computerChoice, attempts).
  • Normalise user input with .toLowerCase() before comparing. Case sensitivity is a common bug.
  • The while loop is ideal for "keep going until the user gets it right" — unknown iteration count.
  • Use continue to skip invalid input and loop again without incrementing the attempts counter.
  • Use return to exit a function early (e.g., when user clicks Cancel on a prompt).
  • Recursion is valid for "play again" logic — calling the game function from within itself, with "no" as the base case.
  • Convert prompt strings to numbers with Number(). Use isNaN() to check if conversion failed.
