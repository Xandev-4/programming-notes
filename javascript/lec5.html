Mastering Loops & Iterations in JavaScript

for loop, while loop, do-while loop, break, continue, nested loops, multiple counters, infinite loops · June 2025

What are Loops?

A loop means performing the same task repeatedly. In programming, instead of writing the same code lines over and over, you use a loop construct — the language automates the repeated execution for you.

● for Loop

Best when you know exactly how many times to loop. Fixed iteration count.

e.g. print numbers 1 to 5

● while Loop

Best when you don't know in advance how many times to loop. Runs as long as a condition is true.

e.g. keep asking for input until user enters correct password

● do-while Loop

Guarantees the loop body runs at least once, regardless of the condition.

e.g. show a menu at least once before checking if user wants to quit
Note An iteration is one cycle of a loop. Looping 5 times = 5 iterations. The word is used interchangeably with "looping" in most documentation.

The for Loop

The for loop is the most used loop. It packs initialization, condition, and update all into one line, making it clean and easy to read for fixed-count loops.

Syntax

for (initialization; condition; update) {
  // code block — runs on each iteration
}
  • Initialization — declares and sets the starting value of the counter variable. Runs once at the very beginning.
  • Condition — checked before every iteration. If true, the block runs. If false, the loop ends.
  • Update — runs after every iteration to move the counter forward (or backward).

How the Engine Executes It (Flowchart in words)

  1. Run initialization once.
  2. Check condition — if false, exit loop immediately.
  3. If true → execute the code block.
  4. Run update.
  5. Go back to step 2.

Example — Print 1 to 5

for (let count = 1; count <= 5; count++) {
  console.log("Iteration / Loop:", count);
}
// Output: 1  2  3  4  5
// count starts at 1, runs while count <= 5, increments by 1 each time
// When count becomes 6, condition 6 <= 5 is false → loop ends

Example — Print in Reverse (Decrement)

for (let i = 5; i >= 1; i--) {
  console.log(i);
}
// Output: 5  4  3  2  1
// Update is i-- (decrement) instead of i++

Example — Sum of Even Numbers from 1 to 100

let sum = 0;

for (let i = 1; i <= 100; i++) {
  if (i % 2 === 0) {   // check if i is even (remainder when divided by 2 is 0)
    sum += i;          // add to running total
  }
}

console.log("Sum:", sum); // Sum: 2550

This is pure logic building: loop through every number 1–100, pick only the even ones, accumulate them. The condition inside the loop (i % 2 === 0) is a direct application of the modulo operator from Lec 3.

Example — Iterate Over a String

Strings in JS are zero-indexed — position 0 is the first character. string.length gives the total character count.

let language = "JavaScript";

for (let i = 0; i < language.length; i++) {
  console.log(language.charAt(i));
}
// charAt(i) returns the character at position i
// Output: J  a  v  a  s  c  r  i  p  t  (one per line)
Note The loop goes from i = 0 to i < language.length (NOT <=) because positions are 0 to 9 for a 10-character string. Using <= would go to position 10 which doesn't exist.

Nested Loops

A nested loop is a loop inside another loop. The inner loop completes all of its iterations for every single iteration of the outer loop. Used when working with multi-dimensional data like tables, matrices, or grids.

How It Executes

  • Outer loop starts: i = 1
  • Inner loop runs completely: j = 1, j = 2, j = 3 → inner loop ends
  • Outer loop increments: i = 2
  • Inner loop runs completely again from scratch: j = 1, j = 2, j = 3
  • Continues until outer loop ends.

Example — Row × Column Grid

for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log("Row:", i, "Column:", j);
  }
}
// Output:
// Row: 1 Column: 1
// Row: 1 Column: 2
// Row: 1 Column: 3
// Row: 2 Column: 1
// Row: 2 Column: 2
// Row: 2 Column: 3
// Row: 3 Column: 1
// Row: 3 Column: 2
// Row: 3 Column: 3

// For each outer iteration (row), the inner loop finishes ALL 3 columns first.

Classic Task — Print a Star Triangle

// Print:
// *
// * *
// * * *
// * * * *
// * * * * *

for (let i = 1; i <= 5; i++) {
  let row = "";
  for (let j = 1; j <= i; j++) {  // inner loop runs i times (grows with outer)
    row += "* ";
  }
  console.log(row);
}
Warning Nested loops have a performance cost. A loop inside a loop = O(n²) time complexity. For large datasets, avoid deep nesting — look for smarter algorithms. Fine for small, fixed data like tables or patterns.

break and continue

Two special control keywords that change the normal flow of a loop. break and continue are often confused — know the difference cold.

● break

Exits the loop immediately. No more iterations happen at all. Execution jumps to the line after the loop's closing brace.

Use: stop looping when a target is found

● continue

Skips the current iteration. Jumps to the update step and starts the next iteration. The loop does NOT end.

Use: skip certain values, process only matching items

● return

Exits the entire function, not just the loop. Implicitly stops the loop too since the function ends.

Use inside functions to exit early

break Example — Stop at 3

for (let i = 1; i <= 5; i++) {
  if (i === 3) break;     // when i is 3, exit loop immediately
  console.log(i);
}
// Output: 1  2
// When i reaches 3, break fires BEFORE console.log runs for 3
// Nothing after break inside that iteration executes
// If console.log comes BEFORE the break:
for (let i = 1; i <= 5; i++) {
  console.log(i);
  if (i === 3) break;
}
// Output: 1  2  3
// Now 3 gets printed before break fires
Tip The position of break relative to other statements in the block matters. Everything before break in that iteration runs; everything after it does not.

continue Example — Skip 3, Print Rest

for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;  // skip iteration when i is 3, move to i = 4
  console.log(i);
}
// Output: 1  2  4  5
// 3 is skipped — but loop continues normally for 4 and 5

break vs continue — Side by Side

// break — exits loop entirely
for (let i = 1; i <= 5; i++) {
  if (i === 3) break;
  console.log(i);         // Output: 1  2
}

// continue — skips one iteration, loop goes on
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);         // Output: 1  2  4  5
}

Multiple Counters in One for Loop

You can declare and update multiple counter variables in a single for loop by separating them with the comma operator (,). Useful when you need two variables moving simultaneously — for example, one going up and another going down. 

// i goes 1 → 10, j goes 10 → 1 simultaneously in ONE loop
for (let i = 1, j = 10; i <= 10 && j >= 1; i++, j--) {
  console.log("i:", i, "j:", j);
}
// Output:
// i: 1  j: 10
// i: 2  j: 9
// i: 3  j: 8
// i: 4  j: 7
// i: 5  j: 6
// i: 6  j: 5
// i: 7  j: 4
// i: 8  j: 3
// i: 9  j: 2
// i: 10 j: 1
  • Initialization: let i = 1, j = 10 — two variables, comma-separated
  • Condition: i <= 10 && j >= 1 — both must be true (AND)
  • Update: i++, j-- — i increments, j decrements simultaneously
Note The comma in initialization (let i = 1, j = 10) is valid inside the for parentheses. The condition section uses && (not comma) because it must be a single boolean expression.

The while Loop

The while loop runs its code block as long as a condition is true. Unlike for, there's no built-in initialization or update — you manage the counter yourself. Use it when you don't know in advance how many times to loop.

Syntax

// Optional: declare counter before loop
let counter = initialValue;

while (condition) {
  // code block

  // Important: update counter here to avoid infinite loop
  counter++;
}

How It Executes (Flowchart in words)

  1. Check condition — if false from the start, the block NEVER runs (zero iterations possible).
  2. If true → execute the code block.
  3. Check condition again — repeat until false.

Example — Print 1 to 5 with while

let count = 1;          // initialization (outside the loop)

while (count <= 5) {    // condition
  console.log(count);
  count++;              // update (inside the loop — must be here!)
}
// Output: 1  2  3  4  5

Comparison with for Loop

// These two are equivalent:

// for loop version:
for (let i = 1; i <= 5; i++) {
  console.log(i);
}

// while loop version (same logic, different structure):
let i = 1;           // initialization
while (i <= 5) {     // condition
  console.log(i);
  i++;               // update
}
Tip A for loop is just a while loop with initialization and update packed into the header. They can always substitute each other — choose whichever is more readable for your use case.

Example — Unknown Iterations (Random Number)

// Keep looping until we randomly get a number > 0.9
// We don't know in advance how many tries that will take — perfect for while

let tries = 0;

while (Math.random() <= 0.9) {
  tries++;
  console.log("Trying... attempt:", tries);
}

console.log("Got a number > 0.9 after", tries, "tries");
// Each run will give different output — could be 1 try, could be 20

Example — Sum Until Limit is Reached

let sum = 0;
let num = 1;

while (sum < 50) {     // keep adding until sum reaches 50 or more
  sum += num;
  num++;
}

console.log("Sum:", sum, "| Stopped at num:", num - 1);
// Output: Sum: 55 | Stopped at num: 10
// We don't know in advance that it takes 10 numbers to exceed 50 → while is ideal

The do-while Loop

The do-while loop is like while, but the condition is checked after the block runs — not before. This guarantees the block executes at least once, no matter what.

Syntax

do {
  // code block — always runs at least ONCE
} while (condition);  // ← note the semicolon after while()
Warning Don't forget the semicolon after while (condition); in a do-while loop. It's easy to miss and will cause a SyntaxError.

Example — Basic do-while

let count = 1;

do {
  console.log(count);
  count++;
} while (count <= 5);
// Output: 1  2  3  4  5
// Same result as a for/while here, but the guarantee is what matters

The Guarantee — Condition False from Start

// while loop — condition false from start, block never runs:
let x = 10;
while (x < 5) {
  console.log("while:", x); // ← NEVER executes
}

// do-while — block runs ONCE even though condition is false:
let y = 10;
do {
  console.log("do-while:", y); // ← Runs ONCE (prints 10)
} while (y < 5);               // condition checked AFTER first run → false → stop
// Output: "do-while: 10"

Real-world Use — Menu System

A do-while is perfect for menus: show the menu at least once, then keep showing it until the user chooses to exit.

// Simulating user menu input:
let choice;

do {
  // In a real app, this would be a prompt() or UI interaction
  choice = Math.floor(Math.random() * 4) + 1; // random 1-4
  console.log("User selected:", choice);
} while (choice !== 4); // 4 = "Exit"
// Menu always shows at least once.
// Keeps looping until user picks 4 (Exit).

do-while vs while — Key Difference

● while

Condition checked before block runs. If condition is false at start, block runs 0 times.

Use when it's OK to skip execution entirely

● do-while

Block runs first, condition checked after. Always runs at least 1 time.

Use when at least one execution is required

● Semicolon!

do-while requires a ; after the closing while(condition). while loop does NOT have a semicolon after the header.

} while (cond); ← required

Infinite Loops

An infinite loop is a loop whose condition never becomes false — so it runs forever. In a browser, this freezes the tab. In Node.js, it hangs the process. Always have an exit condition.

Common Causes

// ❌ Forgetting to update the counter
let i = 1;
while (i <= 5) {
  console.log(i);
  // i++ is missing! i stays 1 forever → infinite loop
}

// ❌ Condition that can never become false
for (let i = 1; i > 0; i++) {
  console.log(i); // i always > 0 (and grows!) → never ends
}

// ❌ Using wrong comparison direction
let count = 10;
while (count >= 1) {
  console.log(count);
  count++;  // count grows! Should be count-- to reach 0
}

Intentional Infinite Loop with break

Sometimes you deliberately write while (true) and rely on break to exit. This is valid and common in certain patterns like game loops or server listeners.

let attempts = 0;

while (true) {             // deliberate infinite loop
  attempts++;
  console.log("Attempt:", attempts);

  if (attempts >= 3) {
    console.log("Max attempts reached. Exiting.");
    break;                 // ← this is the only exit
  }
}
// Output: Attempt: 1  Attempt: 2  Attempt: 3  "Max attempts reached."
Warning If you accidentally run an infinite loop in the browser, the tab will freeze. Open Task Manager (or Activity Monitor on Mac) and kill the browser tab process. Then fix your code.

Which Loop to Use — Decision Guide

The right loop is the one that most clearly expresses your intent. Here's the simple decision rule:

● Use for when...

You know exactly how many iterations you need. You have a definite start, end, and step.

Print 1–100 · iterate over array · repeat N times

● Use while when...

You don't know the iteration count. You loop until an external condition changes.

Read until EOF · retry until success · game loop

● Use do-while when...

You need the loop body to run at least once before checking the condition.

Show menu · initial prompt · validate input once then retry
// Clear examples of the right tool for each job:

// ✅ for — known count (print 1 to 10)
for (let i = 1; i <= 10; i++) {
  console.log(i);
}

// ✅ while — unknown count (loop until random > 0.95)
while (Math.random() <= 0.95) {
  console.log("Rolling...");
}

// ✅ do-while — guaranteed first run (ask at least once)
let answer;
do {
  answer = "yes"; // simulate user input
  console.log("Do you want to continue?", answer);
} while (answer === "yes");

Code Quizzes — Test Your Understanding

Quiz 1 — What is the output?

for (let i = 1; i <= 5; i++) {
  if (i === 3) break;
  console.log(i);
}

Answer: 1 2 — Break fires when i === 3, before console.log gets to run for 3. The loop ends with 1 and 2 printed.

Quiz 2 — What is the output?

for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;
  console.log(i);
}

Answer: 1 2 4 5 — Continue skips iteration 3, but the loop continues from 4 onward. 3 is never printed.

Quiz 3 — What is the output?

let x = 10;

do {
  console.log(x);
  x++;
} while (x < 5);

Answer: 10 — The condition x < 5 is checked AFTER the first run. x = 10 and 10 < 5 is false, so it stops after one iteration. do-while guarantee in action.

Quiz 4 — What is the output?

for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    if (j === 2) break;
    console.log("i:", i, "j:", j);
  }
}

Answer: Only j = 1 prints for each row — i:1 j:1, i:2 j:1, i:3 j:1. The inner break only exits the inner loop — the outer loop continues normally. Break only exits the loop it is directly inside.

Quiz 5 — What is the output and what is the final value of sum?

let sum = 0;
for (let i = 1; i <= 10; i++) {
  if (i % 2 !== 0) continue;  // skip odd numbers
  sum += i;
}
console.log(sum);

Answer: 30 — Sum of even numbers 2 + 4 + 6 + 8 + 10 = 30. Odd numbers (1, 3, 5, 7, 9) are skipped via continue.

Key Takeaways

  • A loop runs a block of code repeatedly without you writing it multiple times.
  • One cycle of a loop = one iteration.
  • for loop — fixed iterations. Syntax: for (init; condition; update) {}.
  • for loop execution order: init → check condition → execute → update → check condition → ...
  • String iteration uses string.length and zero-based indexing: i = 0 to i < length.
  • Nested loops — inner loop completes all iterations for every single outer iteration. Used for 2D data.
  • break — exits the loop immediately. No more iterations. Exits only the loop it's directly inside.
  • continue — skips the current iteration and moves to the next one. Loop does not end.
  • Multiple counters in one for loop: for (let i = 0, j = 10; i <= 10 && j >= 0; i++, j--).
  • while loop — condition checked before each iteration. Block may run 0 times if condition is false initially.
  • do-while loop — block runs first, condition checked after. Guaranteed minimum 1 execution. Note the semicolon.
  • Infinite loop — condition never becomes false. Usually a bug. Always ensure counter updates and exit conditions.
  • while (true) { ... break; } — deliberate infinite loop with break is a valid and common pattern.
