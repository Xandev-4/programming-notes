Mastering Control Flow in JavaScript

if, if-else, else-if chains, nesting, switch-case, fall-through, ternary, and when to use what · June 2025

What is Control Flow?

By default, JavaScript executes code line by line, top to bottom. But real programs need to make decisions — skip certain lines, jump to others, repeat blocks. Control flow is the ability to direct which lines execute, in what order, based on conditions.

● Sequential

Default mode. JS executes every line one after another, no skipping.

Line 1 → Line 2 → Line 3 → ...

● Branching

Based on a condition, execution jumps to one branch or another — skipping the rest.

if / else / switch-case

● Keywords

Special words that control how far execution travels inside a branch or loop.

break · continue · return
Note Control flow is what turns a list of instructions into actual logic. Without it, every program would just run all lines always — useless for real-world problems.

The if Statement

The if keyword takes a condition in parentheses. If the condition evaluates to truthy, the block inside curly braces {} executes. If falsy, the block is skipped entirely.

Syntax

if (condition) {
  // code to run if condition is truthy
}

Example — Catching a Bus

let catchingBus = true;

if (catchingBus) {
  console.log("I will reach home on time.");
}
// Output: "I will reach home on time."
// If catchingBus were false, nothing would print at all.

Example — Voting Eligibility

let age = 18;

if (age >= 18) {
  console.log("You are eligible to vote.");
}
// age >= 18 → 18 >= 18 → true → block executes
// Output: "You are eligible to vote."
Note The condition inside if () must resolve to true or false. You can use any comparison or logical operator here — that's exactly why we learned them in Lec 3.

Omitting Curly Braces — Don't

If your if block has only ONE statement, curly braces are technically optional. But always use them anyway. Omitting braces is a common source of subtle bugs.

// Technically works (only one statement):
if (age >= 18)
  console.log("Eligible to vote.");

// Recommended — always use braces:
if (age >= 18) {
  console.log("Eligible to vote.");
}

// Why it matters — without braces, only the FIRST line belongs to if:
if (age >= 18)
  console.log("Eligible to vote.");
  console.log("Go to the voting center."); // ← This runs ALWAYS, not part of if!
Warning Always use curly braces {}. Never omit them, even for single-line blocks. Indentation is misleading — only braces determine what belongs to the block.

The if-else Statement

When you need to do one thing if a condition is true and something else if it's false, add an else block. The else runs when — and only when — the if condition is falsy.

Syntax

if (condition) {
  // runs if condition is truthy
} else {
  // runs if condition is falsy
}

Example — Voting Eligibility with else

let age = 8;

if (age >= 18) {
  console.log("You are eligible to vote.");
} else {
  console.log("You are NOT eligible to vote.");
}
// age is 8 → 8 >= 18 is false → else block runs
// Output: "You are NOT eligible to vote."

How JS Executes if-else

JS doesn't execute both blocks — it executes exactly one. Once the condition is evaluated:

  • If truthy → enter if block, skip else completely.
  • If falsy → skip if block, enter else block.
Note else cannot exist without an if. Writing a standalone else throws a SyntaxError: Unexpected token 'else'. They are always a pair.

else-if Chain (Multiple Conditions)

When you have more than two possible outcomes, chain multiple conditions using else if. JS evaluates them top to bottom and enters the first matching block, then skips all remaining ones.

Syntax

if (condition1) {
  // runs if condition1 is true
} else if (condition2) {
  // runs if condition1 was false AND condition2 is true
} else if (condition3) {
  // runs if condition1 and condition2 were false AND condition3 is true
} else {
  // runs if ALL conditions above were false
}

Example — Grading System

let score = 76;

if (score >= 90) {
  console.log("Grade A");
} else if (score >= 80) {
  console.log("Grade B");
} else if (score >= 70) {
  console.log("Grade C");
} else {
  console.log("Fail");
}
// score = 76 → not >= 90 → not >= 80 → IS >= 70 → "Grade C"
// Once "Grade C" matches, else (Fail) is NEVER checked.
Tip The key power of else if is early exit. Once a condition matches, JS does not evaluate the remaining branches at all. This makes it more efficient than multiple independent if blocks.

Independent ifs vs else-if — Critical Difference

These two look similar but behave very differently. Choose based on your use case.

let x = 0;

// ── INDEPENDENT if blocks — ALL conditions are checked ──
if (x === 0)        console.log("zero");         // ✅ runs (x is 0)
if (x >= 0)         console.log("greater than or equal to zero"); // ✅ runs (0 >= 0)
if (x <= 0)         console.log("less than or equal to zero");    // ✅ runs (0 <= 0)
// Output: "zero", "greater than or equal to zero", "less than or equal to zero"
// All three run because each if is checked independently.

// ── else-if chain — STOPS at first match ──
if (x === 0) {
  console.log("zero");                          // ✅ runs — matches!
} else if (x >= 0) {
  console.log("greater than or equal to zero"); // ⏭️ SKIPPED — first already matched
} else if (x <= 0) {
  console.log("less than or equal to zero");    // ⏭️ SKIPPED
}
// Output: "zero" — only the first match runs, rest are ignored.
Tip Use independent if blocks when multiple conditions can be true at the same time and you want all matching branches to run. Use else-if chains when only one outcome should happen and you want early exit.

Nested if-else

You can place an if-else inside another if-else block. This is called nesting. It allows you to check a secondary condition only after a primary condition passes. 

let condition = true;
let innerCondition = false;

if (condition) {
  console.log("Outer if");       // condition is true → enters here

  if (innerCondition) {
    console.log("Inner if");     // innerCondition is false → skipped
  } else {
    console.log("Inner else");   // ← this runs
  }

} else {
  console.log("Outer else");     // skipped because outer condition was true
}

// Output:
// "Outer if"
// "Inner else"
// If BOTH conditions are false:
let condition = false;
let innerCondition = false;

if (condition) {
  // Entire outer if block is skipped — inner if/else never even evaluated
  if (innerCondition) {
    console.log("Inner if");
  } else {
    console.log("Inner else");
  }
} else {
  console.log("Outer else");  // ← only this runs
}
// Output: "Outer else"
Tip Avoid nesting more than 2–3 levels deep. Deep nesting makes code hard to read and debug. If you find yourself nesting many levels, it's usually a sign to refactor into functions or use early returns.

switch-case Statement

The switch statement is designed for checking a single value against multiple fixed cases. Instead of writing many else if blocks, you list out cases cleanly. It uses strict equality (===) to match values.

Syntax

switch (value) {
  case option1:
    // code if value === option1
    break;
  case option2:
    // code if value === option2
    break;
  default:
    // code if no case matched (like else)
}

Example — Position to Rank

let position = 4;

switch (position) {
  case 1:
    console.log("Print One");
    break;
  case 2:
    console.log("Print Two");
    break;
  case 3:
    console.log("Print Three");
    break;
  case 4:
    console.log("Print Four");   // ← position is 4, this matches
    break;
  default:
    console.log("Nothing matched");
}
// Output: "Print Four"

The default Case

Like else in an if-chain, default runs when no case matches. It is optional but recommended.

let position = 51;

switch (position) {
  case 1:  console.log("One");  break;
  case 2:  console.log("Two");  break;
  default: console.log("Nothing matched");  // ← runs for 51
}
// Output: "Nothing matched"

Example — Day of the Week

let day = 5;

switch (day) {
  case 1:  console.log("Monday");    break;
  case 2:  console.log("Tuesday");   break;
  case 3:  console.log("Wednesday"); break;
  case 4:  console.log("Thursday");  break;
  case 5:  console.log("Friday");    break;  // ← day is 5, matches here
  case 6:  console.log("Saturday");  break;
  case 7:  console.log("Sunday");    break;
  default: console.log("Invalid day number");
}
// Output: "Friday"

switch with String Values

Cases aren't limited to numbers — they can be any value. Matching is strict (case-sensitive for strings).

let name = "tapaScript";

switch (name) {
  case "tapaScript":
    console.log("Teaching 40 Days of JS");
    break;
  case "Google":
    console.log("Giving answers to all searches");
    break;
  default:
    console.log("You are neither Google nor tapaScript");
}
// Output: "Teaching 40 Days of JS"

// Note: "google" (lowercase g) would NOT match "Google"
// switch uses === — strict equality — so casing matters

The break Keyword

break tells JavaScript to exit the switch block immediately after a case matches. Without break, JS continues executing the code of every subsequent case — even ones that don't match. This is called fall-through.

Without break — Fall-Through Problem

let position = 1;

switch (position) {
  case 1:
    console.log("Print One");   // ← matches, runs
    // NO break here!
  case 2:
    console.log("Print Two");   // ← also runs! (fall-through)
  case 3:
    console.log("Print Three"); // ← also runs! (fall-through)
  case 4:
    console.log("Print Four");  // ← also runs! (fall-through)
}
// Output: "Print One", "Print Two", "Print Three", "Print Four"
// NOT what we wanted — position was 1!

With break — Correct Behaviour

let position = 1;

switch (position) {
  case 1:
    console.log("Print One");   // ← matches, runs
    break;                      // ← exits switch immediately
  case 2:
    console.log("Print Two");   // skipped
    break;
  case 3:
    console.log("Print Three"); // skipped
    break;
}
// Output: "Print One" — only one case runs ✅
Warning Always add break at the end of every case block unless you are intentionally using fall-through. Missing a break is one of the most common switch-case bugs.

Intentional Fall-Through

Sometimes fall-through is useful on purpose. If multiple cases should run the same code, you can stack them without break between them. Only the last case in the group needs the break. 

let day = 6;

switch (day) {
  case 1:
  case 2:
  case 3:
  case 4:
  case 5:
    console.log("Weekday");   // Cases 1–5 all fall through to here
    break;
  case 6:
  case 7:
    console.log("Weekend");   // Cases 6–7 both fall through to here
    break;
  default:
    console.log("Invalid day");
}
// day is 6 → matches case 6 → falls through to case 7's block → "Weekend"
// Output: "Weekend"
Tip Intentional fall-through is a clean way to group cases that share the same outcome. It's much better than duplicating the same console.log in five separate case blocks.

if-else vs switch-case — When to Use What

Both achieve branching, but they are optimised for different problems. Choosing the right one makes your code faster and more readable.

● Use if-else when...

You have complex logical conditions — ranges, multiple variables, compound expressions.

e.g. score >= 80 && score < 90, age > 10 && age < 20

● Use switch-case when...

You're checking a single variable against multiple fixed values — numbers, strings, booleans.

e.g. position = 1/2/3/4, day = "Mon"/"Tue"

● Performance

switch uses a jump table — values are indexed, so lookups are O(1). if-else chains are sequential O(n).

switch wins for many conditions on a fixed value

The Jump Table Explained

When JS compiles a switch, it builds an internal lookup table — a map of all case values to their code blocks. When execution hits the switch, it looks up the value in the table directly and jumps straight to the right case. No sequential checking.

With if-else, JS checks each condition one by one: is it this? No. Is it this? No. Is it this?... — sequential. For 10+ conditions, this gap is meaningful.

// Equivalent logic — different tool choices:

// ✅ Use if-else: complex logical conditions (ranges)
if (score >= 90) {
  console.log("Grade A");
} else if (score >= 80) {
  console.log("Grade B");
} else if (score >= 70) {
  console.log("Grade C");
} else {
  console.log("Fail");
}

// ✅ Use switch-case: fixed discrete values
switch (dayNumber) {
  case 1: console.log("Monday");    break;
  case 2: console.log("Tuesday");   break;
  case 3: console.log("Wednesday"); break;
  // etc.
}
Note Rule of thumb: range / complex condition? → if-else. Single fixed value check? → switch-case. When in doubt about performance with many cases, prefer switch.

Ternary Operator in Control Flow

The ternary operator (condition ? a : b) is a concise single-expression alternative to a simple if-else. It's great for short, clear decisions — especially when assigning a value based on a condition. 

// if-else version:
let age = 20;
let access;
if (age >= 18) {
  access = "Allowed";
} else {
  access = "Denied";
}
console.log(access); // "Allowed"

// Ternary version — same logic, one line:
let access = age >= 18 ? "Allowed" : "Denied";
console.log(access); // "Allowed"
// Another example:
let score = 45;
let result = score >= 50 ? "Pass" : "Fail";
console.log(result); // "Fail"
Tip Use ternary for simple, one-condition assignments. Avoid chaining ternaries — a ? b : c ? d : e — it becomes unreadable fast. Reach for if-else when logic is complex.

Code Quizzes — Test Your Understanding

Work through these mentally before checking the answers. These are exactly the kind of questions asked in interviews.

Quiz 1 — What is the output?

let x = 0;

if (x === 0)  console.log("A");
if (x >= 0)   console.log("B");
if (x <= 0)   console.log("C");

Answer: A, B, C — all three print. These are independent if blocks, not an else-if chain. All three conditions are true when x = 0.

Quiz 2 — What is the output?

let x = 0;

if (x === 0) {
  console.log("A");
} else if (x >= 0) {
  console.log("B");
} else if (x <= 0) {
  console.log("C");
}

Answer: Only A. This is an else-if chain. The first condition matches (x === 0), so B and C are never checked.

Quiz 3 — What is the output?

let condition = false;
let innerCondition = true;

if (condition) {
  if (innerCondition) {
    console.log("Inner if");
  } else {
    console.log("Inner else");
  }
} else {
  console.log("Outer else");
}

Answer: Outer else. Since condition is false, the entire outer if block — including the nested if/else — is skipped entirely. innerCondition is never even evaluated.

Quiz 4 — What is the output?

let position = 2;

switch (position) {
  case 1:
    console.log("One");
  case 2:
    console.log("Two");
  case 3:
    console.log("Three");
  default:
    console.log("Default");
}

Answer: Two, Three, Default. There are no break statements — so after matching case 2, JS falls through into case 3 and default as well.

Key Takeaways

  • Control flow = controlling which lines of code execute based on conditions.
  • if (condition) {} — runs block if condition is truthy. Skips if falsy.
  • else {} — runs when the if condition is falsy. Cannot exist without if.
  • else if — chains multiple conditions. Stops at the first match. Use for mutually exclusive outcomes.
  • Independent if blocks = all conditions checked. else-if = early exit on first match.
  • Always use curly braces {} — even for single-line if blocks.
  • Nesting if-else is valid but avoid going deeper than 2–3 levels.
  • switch (value) — checks a single value against multiple fixed case options using strict equality ===.
  • break is mandatory after each case — without it, execution falls through into every subsequent case.
  • Intentional fall-through — stack multiple cases without break to share the same code block.
  • default — the fallback case in switch, like else in if-chains.
  • if-else = complex conditions, ranges, multiple variables. switch-case = single fixed value, many options.
  • switch uses a jump table internally — faster than sequential if-else for many conditions.
  • Ternary condition ? a : b — concise if-else for simple single-condition value assignments.
